# 当科研任务卡住时：把“我不会”改写成可执行的小问题

## 0. 写在前面

我在科研工作中经常遇到一种状态：

> 文献读不懂，公式看不懂，PPT 做不清楚，任务推进很慢。
> 然后我开始不耐烦，打开无关窗口，或者转向“安装工具”“整理环境”这种看起来像工作的事情。

表面上看，这是“努力不够”或“自控力差”。
但更准确地说，很多时候是因为任务没有被定义成一个可执行的小问题。

例如：

* “读懂这篇文献”太大；
* “搞懂这个公式”太大；
* “完善 PPT”太大；
* “找一个研究方向”也太大。

这些任务的问题是：没有明确边界，没有完成标准，也没有可见产物。
所以它们很容易变成无底洞。

这篇笔记记录的是一个可复用原则：

> 当科研任务卡住时，不要继续问“我怎样才能全部搞懂”，而要问：
> **我现在卡在哪一层？我能否把它改写成一个 45–70 分钟内可完成、可记录、可停止的小任务？**

---

# 1. 总原则：工作块必须留下外部产物

## 1.1 什么是不合格任务

下面这些任务看起来合理，但实际很容易失控：

```text
读文献
理解公式
完善 PPT
搞懂机制
总结领域
找研究方向
```

它们的问题是：都指向一个脑内状态。

比如“理解公式”，到底怎样算理解？
是知道符号？知道输入输出？能讲物理意义？还是能逐行推导？

如果没有定义，任务就会无限膨胀。

---

## 1.2 什么是合格任务

合格的工作块任务必须留下外部产物。

也就是说，工作结束时，我要能指着一个东西说：

> 这就是这 45–70 分钟留下的结果。

合格产物可以是：

```text
一张文献五句卡
一行“现象—机制—证据图”表格
一张黑箱公式卡
一页 PPT 的三句话讲稿
一个推导断点记录
一份代码运行日志
一张测试图
```

核心区别是：

| 不合格目标  | 合格产物         |
| ------ | ------------ |
| 读文献    | 写出文献五句卡      |
| 理解公式   | 写出黑箱公式卡      |
| 完善 PPT | 写出一页三句话讲稿    |
| 搞懂机制   | 记录机制链条和断点    |
| 找方向    | 写出一个候选研究问题卡  |
| 复现代码   | 保存一次运行记录或测试图 |

---

## 1.3 工作块模板

以后每次开始工作前，先写这个：

```markdown
## 工作块

时间：

对象：

动作：

产物：

停止条件：

如果卡住：
```

示例：

```markdown
## 工作块

时间：15:30–16:20

对象：文献 A 的 Fig. 2

动作：提取现象、机制和证据链

产物：
完成一行“现象—机制—证据图”表格

停止条件：
填完三列即停止，不要求推懂所有公式

如果卡住：
把公式黑箱化，只记录：
1. 它计算什么物理量；
2. 输入和输出是什么；
3. 它解释哪张图或哪个现象。
```

# 2. 卡点一：文献读不懂

## 2.1 常见错误

我容易把目标设成：

> 读懂这篇文献。

但“读懂”太抽象。
一篇文章里有背景、模型、公式、图、结果、讨论和补充材料。如果一开始就要求全部读懂，很容易卡住。

---

## 2.2 最低产物：文献五句卡

读一篇文献的最低产物不是“我懂了”，而是能写出五句话：

```markdown
## 文献五句卡

文献：

1. 这篇文章研究的问题是：

2. 研究体系 / 对象是：

3. 作者观察到的核心现象是：

4. 作者给出的机制解释是：

5. 最关键的证据图是：
```

如果这五句话写不出来，说明这篇文章还没有进入可汇报状态。

但五句话不要求完美。
可以写成：

```text
作者认为该现象与 xxx 机制有关，但我还没有理解公式 X 如何推出这个结论。
```

这仍然是有效进展，因为它明确记录了理解边界。

---

## 2.3 更进一步：现象—机制—证据图

如果要准备组会或 PPT，可以把文献压缩成一行表格：

```markdown
| 文献 | 现象 | 机制 | 证据图 | 当前理解等级 |
|---|---|---|---|---|
|  |  |  |  |  |
```

其中“当前理解等级”可以写：

```text
现象理解；机制半理解；公式未推懂。
```

这能防止我把自己没有理解的内容默认成“已经懂了”。

---

## 2.4 Takehome message

> 读文献的第一产物不是“读完”，而是写出：
> **问题、体系、现象、机制、证据图。**

---

# 3. 卡点二：公式和机制看不懂

## 3.1 常见错误

当文献说：

> 某个效应来源于某个公式。

我看不懂时，容易直接问：

> 这个公式怎么推导？

但这个问题经常太大。
AI 可能给出一长串推导，而我仍然看不懂。

原因是：我可能并不是卡在“推导”本身，而是卡在更前面的层级。

---

## 3.2 先判断：我卡在哪一层？

看不懂公式时，先不要推导。
先判断自己卡在哪一层：

| 层级    | 典型问题                   |
| ----- | ---------------------- |
| 符号层   | 每个符号是什么意思？             |
| 物理量层  | 这个量代表什么物理意义？           |
| 公式类型层 | 这是定义式、响应公式、微扰公式还是有效模型？ |
| 数学步骤层 | 为什么这一行能变成下一行？          |
| 近似条件层 | 为什么可以忽略某些项？            |
| 机制层   | 这个公式为什么能解释该现象？         |
| 证据链层  | 它如何支持图和结论？             |

---

## 3.3 分层提问模板

### 第一问：符号层

```text
请只解释这个公式中每个符号的定义、维度、单位和物理意义，不要推导。
如果某个符号在文中有特殊定义，请指出它出现在哪个上下文中。
```

如果符号层还不清楚，就不要进入推导。

---

### 第二问：公式类型层

```text
请判断这个公式属于哪一类公式：
定义式、响应公式、微扰论公式、有效哈密顿量、半经典方程、拓扑量、经验拟合，还是其他？

不要推导，只说明：
1. 它计算什么物理量；
2. 输入是什么；
3. 输出是什么；
4. 适用条件是什么；
5. 它在文章逻辑中起什么作用。
```

---

### 第三问：断点定位层

如果已经有推导，但某一步看不懂，不要要求“重新推一遍”。

改成：

```text
下面是一个推导。我只能理解到第 [n] 行。

请只解释第 [n] 行到第 [n+1] 行：
1. 这一步用了什么定义？
2. 这一步用了什么数学变换？
3. 这一步用了什么物理假设或近似？
4. 是否省略了中间步骤？
5. 这一步的物理意义是什么？

不要继续解释后面的推导。
```

---

## 3.4 如果短时间推不懂：黑箱公式

不是所有公式都必须当场推懂。
短时间推不懂时，可以先黑箱化。

但要区分两种情况。

### 逃避式跳过

```text
看不懂公式
→ 跳过
→ PPT 照放
→ 被问到答不上来
```

### 研究式黑箱

```text
看不懂公式
→ 记录它计算什么物理量
→ 记录输入和输出
→ 记录它解释哪张图或哪个现象
→ 标记当前不懂的位置
→ 决定它放主线、备份页，还是不放
```

---

## 3.5 黑箱公式卡

```markdown
## 黑箱公式卡

公式编号：

公式计算的物理量：

输入：

输出：

它解释的图 / 现象：

它在文章逻辑中的作用：

我目前理解到：

我没有理解：

PPT 处理方式：
- [ ] 主线讲
- [ ] 放备份页
- [ ] 暂时不放

下一步只补：
-
```

---

## 3.6 Takehome message

> 看不懂公式时，不要直接问“怎么推导”。
> 先判断自己卡在符号、物理量、公式类型、数学步骤、近似条件、机制还是证据链。
> 如果短时间推不懂，可以黑箱化，但必须记录输入、输出、计算对象和解释对象。

# 4. 卡点三：PPT 有内容但讲不清

## 4.1 常见错误

我容易以为：

> PPT 上有内容 = 我可以讲。

但组会真正暴露的问题往往是：

* 这一页为什么存在？
* 这张图支持什么结论？
* 这个公式和图有什么关系？
* 如果导师追问一步，我能不能回答？

所以 PPT 的最小单位不是 slide，而是这页背后的讲述和追问防线。

---

## 4.2 一页 PPT 的三句话讲稿

每一页 PPT 至少要有三句话：

```markdown
## PPT 页面卡

页码：

本页核心结论：

这页我要讲的三句话：
1.
2.
3.

本页证据：

导师可能追问：
1.
2.
3.

我能回答：

我不能回答：

处理方式：
- [ ] 保留在主线
- [ ] 改成备份页
- [ ] 删除
- [ ] 需要补一张图 / 一句话解释
```

如果一页 PPT 写不出三句话，说明它可能还不适合放进主线。

---

## 4.3 PPT 内容取舍规则

一个图或公式是否放进 PPT 主线，可以用这个标准判断：

| 理解状态           | PPT 处理 |
| -------------- | ------ |
| 能讲清现象、机制、证据链   | 放主线    |
| 只能说明作用，但推导不熟   | 放备份页   |
| 连输入输出和解释对象都说不清 | 暂时不放   |
| 只是因为“文献里有”才想放  | 谨慎删除   |

---

## 4.4 Takehome message

> PPT 上的每张图、每个公式，都必须有最低追问防线。
> 如果我不能说明它证明什么，就不要把它放进主线。

# 5. 卡点四：任务太大，开始逃避

## 5.1 逃避的常见触发

我经常不是一开始就不想工作，而是在这些情况下开始逃避：

* 任务没有明确下一步；
* 预计需要很长时间；
* 公式推不动；
* 机制看不懂；
* PPT 感觉永远改不完；
* 不知道做到什么程度算完成。

这时我容易转向：

* 玩游戏；
* 打开无关窗口；
* 查工具；
* 安装软件；
* 整理环境；
* 做一些“看起来像工作”的事情。

---

## 5.2 把目标降级：从“解决问题”到“定位断点”

当一个问题太大时，不要要求自己立刻解决它。
先把目标降级为：

> 定位第一个断点。

断点记录模板：

```markdown
## 断点记录

任务：

我原本想解决的问题：

我现在能理解到：

第一个卡住的位置：

它属于哪类断点：
- [ ] 符号不懂
- [ ] 物理量不懂
- [ ] 公式类型不懂
- [ ] 数学步骤不懂
- [ ] 近似条件不懂
- [ ] 物理机制不懂
- [ ] 证据链不懂

下一步只做：
```

---

## 5.3 区分休息和逃避

我确实需要休息，也需要自由支配时间。
但要区分主动休息和逃避摸鱼。

### 主动休息

离开前我能说清：

```text
我刚刚完成了什么；
我休息多久；
休息后回来做什么。
```

### 逃避摸鱼

离开前通常是：

```text
太烦了；
这个没完没了；
先不做了；
不做就不做吧；
再做也只会拖更久。
```

---

## 5.4 Takehome message

> 如果解决不了问题，就先定位断点。
> 断点定位本身也是工作产物。
> 自由时间应该通过规划获得，而不是通过逃避夺回。

---

# 6. 下次卡住时的快速流程

以后遇到卡点时，不再笼统地说“我不会”。
按这个流程走：

```text
1. 我现在面对的对象是什么？
   - 文献 / 公式 / 图 / PPT / 代码 / 方向

2. 我卡在哪一层？
   - 符号 / 物理量 / 公式类型 / 数学步骤 / 近似条件 / 机制 / 证据链

3. 这个工作块的外部产物是什么？
   - 卡片 / 表格 / 讲稿 / 断点记录 / 图 / 运行日志

4. 45–70 分钟内能否完成？
   - 能：开始
   - 不能：继续缩小

5. 如果不能搞懂，是否可以黑箱化？
   - 记录输入、输出、计算对象、解释对象

6. 这个内容是否应该进 PPT 主线？
   - 能讲清：主线
   - 只能说明作用：备份页
   - 连作用都讲不清：暂时不放
```

# 7. 最重要的几个 Takehome Messages

## 1. 模糊任务会制造逃避

如果任务写成“读文献”“理解公式”“完善 PPT”，它很容易失控。
任务必须改写成可完成的外部产物。

---

## 2. 看不懂不是终点

看不懂之后要继续问：

> 我到底卡在哪一层？

不是所有“不懂”都是同一种不懂。

---

## 3. 黑箱不是问题，无记录黑箱才是问题

如果短时间推不懂公式，可以黑箱化。
但必须记录：

* 它计算什么；
* 输入是什么；
* 输出是什么；
* 它解释什么图或现象。

---

## 4. PPT 的核心不是放内容，而是能回答追问

每一页都要有：

* 三句话讲稿；
* 核心证据；
* 可能追问；
* 当前不能回答的部分。

---

## 5. 工作块的目标不是“全部搞懂”，而是留下下一步可以接上的记录

一次 45–70 分钟的工作块，只需要留下：

* 一张卡；
* 一行表；
* 一个断点；
* 一页讲稿；
* 一份运行记录。

这些都是有效进展。

---

# 8. 一句话总结

科研工作中的很多“努力问题”，本质上是任务没有被拆成可执行、可记录、可停止的小问题。

我的目标不是立刻全部搞懂，而是持续把模糊卡点转化成：

> 明确对象 → 明确断点 → 明确产物 → 明确下一步。
